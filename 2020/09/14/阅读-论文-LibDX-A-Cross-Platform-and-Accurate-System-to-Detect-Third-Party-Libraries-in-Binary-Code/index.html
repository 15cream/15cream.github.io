[阅读|论文]LibDX: A Cross-Platform and Accurate System to Detect Third-Party Libraries in Binary Code

BibTeX:

这是BCFinder的姊妹篇，或者说是进阶篇。

阅读笔记

摘要

背景和问题

  1. 开源盛行，第三方库使用频繁；
  2. 第三方库误用带来的风险：license violation and security vulnerability；
  3. 识别C和C++开发的第三方库的挑战：编译过程隐藏了大多数的代码特征，且不同的编译过程对最终形成的binary影响很大。

解决方案

  1. 提出了LibDX，可以从binaries中检测出重用的第三方库；
  2. 提取的特征能够克服compilation diversity，因此LibDX具有platform-independent属性；
  3. 提出了一种处理feature duplication的方法：（原文）LibDX novelly introduces the logic feature block concept which is applied to deal with the feature duplication challenge in a large-scale feature database.

评估

  1. 测试集：覆盖多个平台，9.5K packages including 25.8K C/C++ binary files；
  2. precision： 92%， recall： 97%；
  3. 使用封闭源商业应用程序验证了LibDX的性能，并发现了一些违反许可证的情况。
