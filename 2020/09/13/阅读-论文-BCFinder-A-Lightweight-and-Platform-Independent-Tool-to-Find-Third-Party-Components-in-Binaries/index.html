[阅读|论文]BCFinder: A Lightweight and Platform-Independent Tool to Find Third-Party Components in Binaries

BibTeX:

阅读笔记

摘要

背景和问题

  1. 开源盛行；
  2. CBD(component-based development)和代码重用频繁；
  3. 以上背景引入了一些问题，例如license violation和security weaknesses；
  4. （原文）While code reuse detection has been extensively studied in source form, third-party components detection for software in binary form especially based on large scale database like Github has been less researched.

解决方案

  1. 采集了22K C/C++ repos on Github；
  2. Extend the code reuse detection for binaries against such a large-scale data set；
  3. 设计了一个系统BCFinder，可以通过特征匹配(feature matching)自动识别二进制文件中的第三方库组建。

评估

  1. 在大量真实的程序二进制文件上使用BCFinder，这些binaries是跨平台的，且编译选项不同（？）；
  2. 实验结果显示BCFinder是一个有效的二进制分析辅助工具；

总结

（原文）BCFinder is, by far, the first lightweight, rapid and platform-independent tool to detect component reuse in binaries against a large-scale data base like Github.

重点内容（个人偏向）

challenges

首先作者选择了feature matching作为third-party components detection的方法，且选择的feature具体为string constants。此时会遇到两个challenges。

Partially built binaries.
  1. 第三方组件中只有部分代码被包含到待测二进制中；
  2. 导致低匹配占比（a low matching proportion），进而导致false negatives；
  3. 对问题的定性：（原文）This is caused by an innate problem, for lacking of features.
  4. 解决方案：（原文）To identify incomplete components, we use reserved unique strings (strings appear only one time in all repos) in binaries to determine the existence of a component.
String duplication.
  1. 从binaries中提取的strings可能重复出现在不同的repos中，导致false positives；
  2. 解决方案：（原文）We have taken a series of measures to deduplicate from building the database to feature matching.

Strings Duplication and Deduplication

作者将Strings Duplication分为两类：

  1. Internal code clones in components. 即整个模块的包含，例如有三个模块都包含了第四个模块，导致第四个模块的unique strings出现在前三个模块中。作者会尝试识别出这样的情况。
  2. Cross-repetition between components. 简单的component之间存在重复的strings？作者会忽略在不同components之间出现次数超过16次的strings。

Matching method

  1. 对unique strings和duplicated strings区分对待。当binary与一个component的matched unique strings 和 matched duplicated strings 占比分别达到某个threshold时（会有两个thresholds），认为该component是有效的。
  2. （原文）After matching repos list is generated, we apply a post-process which deletes repos with low matching rate. From the last one in the matching list, we check one by one from back to front. If the majority of matched strings of a repo can be included in repos with higher matching rate, we think this repo is false positive as a result of strings duplication. Finally, we only show matching results with high matching ratio and high absolute matching quantity.

总结

  1. 可以了解一些Third-party component detection的背景知识，例如它在software security, commercial profit, software maintenance, and copyright protection这些方面的作用；
  2. 可以了解下作者对组件间相似问题的定义和解决方案。
